# PerioNote Bootstrap Spec

## Purpose

PerioNote is a single repository that supports:

1. a **template library** for reusable periodontal / hygiene note templates
2. a **preview app** for rendering, testing, and manually using templates
3. a **migration workspace** for importing standalone JSX templates previously generated by ChatGPT
4. a **Codex-friendly workflow** for safely proposing, reviewing, and merging AI-assisted code changes

The goal is to let contributors work from desktop or on the go, while keeping templates easy to preview, import, test, and refine over time.

---

## Core Product Direction

PerioNote should be **both**:

- a **library of reusable TSX template components**
- a **runnable Next.js app** that previews those templates inline

This dual structure allows:
- fast import of existing standalone JSX templates
- reusable components for future templates
- browser-based preview for iPad/MacBook use
- automated preview deploys
- local development in VS Code
- Codex-assisted iteration in GitHub

---

## Stack

### Primary stack
- **Next.js**
- **TypeScript**
- **Tailwind CSS**

### Testing
- **Vitest** for unit and fixture-based tests
- **Playwright** for preview-page smoke tests
- **Playwright full E2E** only for important/stable templates

### Git hosting
- **GitHub**

### AI workflow
- **Codex**
- wrapping imported templates in `.jsx` or `.tsx` components
      gingival-description-webform/

---

## Why Tailwind-only

PerioNote should start with **Tailwind only**.

Reasons:
- quickest path for importing legacy ChatGPT JSX files
- fewer dependencies and less setup friction
- easier to clean up inconsistent generated templates later
- more predictable for Codex and ChatGPT-generated UI output

Optional libraries like `lucide-react` may still be used for icons, but UI structure should not depend on component libraries.

---

## Repository Goals

The repository should support all of the following:

- previewing templates in the browser
- importing legacy JSX files quickly
- wrapping imported templates in TSX pages
- extracting reusable logic over time
- generating polished clinical summaries
- testing formatter output against fixtures
- reviewing changes through pull requests
- using Codex in a controlled branch workflow
- deploying preview builds automatically
